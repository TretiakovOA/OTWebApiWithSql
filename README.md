Реализовать Web.API приложение для кредитных заявок и заведения нового пользователя  в системе с использованием RestAPI и трёхслойной архитектуры (интерфейс логики, имплементация интерфейса логики, интерфейс доступа к базе, имплементация интерфейса доступа к базе)

Реализовать CRUD операции для следующих сущностей:

1. Клиент (Минимальный набор полей: id, ФИО, Возраст, Текущее место работы, телефон)
2. Кредитная заявка (Минимальный набор полей: id, Причина займа, Объем займа, Доход клиента, id кредитного продукта)
3. Кредитные продукты (Минимальный набор полей: id, Наименование кредитного продукта, Процентная ставка)
4. Звонок (Минимальный набор полей: id, Дата назначения звонка, Результат общения с клиентом, Обработан ли звонок)

Требования: после подачи кредитной заявки от клиента создается объект звонка (реализовывать отдельным сервисом и WEB.API не требуется)
UI реализация не требуется.
*(задание со звёздочкой)
Реализовать дополнительную сущность, в которой было бы реализовано отношение многие ко многим. Пример, который можно использовать: (операторы и ответственность за звонки)
Код написать на C# и SQL (без использования EF или ORM), а так же без использования сторонних nuget пакетов.
